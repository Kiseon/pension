# New User Onboarding

신규 사용자의 목표는 가능한 적은 마찰로 첫 번째 신뢰 가능한 노후 수입
프로젝션을 얻는 것이다.

## Onboarding principles

- 사용자가 모르는 값은 추정값으로 입력할 수 있게 하되, 결과에서 신뢰도 차이를 표시한다.
- 입력 단계마다 왜 필요한 정보인지 설명한다.
- 배우자 정보는 선택 가능하지만, 부부 가구의 프로젝션 품질에는 큰 영향을 준다는 점을 안내한다.
- 민감한 자산 정보는 저장 여부와 삭제 방법을 명확히 고지한다.

## Flow

### 1. Household setup

입력 항목:

- 본인 생년월일
- 배우자 생년월일 또는 배우자 없음
- 거주 국가 및 기준 통화
- 분석 시작 월
- 분석 종료 연령 또는 기대 수명

검증:

- 생년월일이 미래가 아닌지 확인한다.
- 분석 종료 월이 시작 월보다 뒤인지 확인한다.
- 배우자 생년월일이 입력된 경우 두 사람의 월별 연령을 독립적으로 계산한다.

### 2. Retirement goals

입력 항목:

- 본인 은퇴 예정 연령
- 배우자 은퇴 예정 연령
- 희망 월 생활비
- 필수 생활비와 선택 지출의 구분

결과 활용:

- 수입 공백 구간을 표시한다.
- 희망 생활비 대비 수입 충족률을 계산한다.

### 3. Real estate assets

입력 항목:

- 자산 종류: 거주 주택, 임대 주택, 상가, 토지, 기타
- 현재 추정 가치
- 보유 지분율
- 월세 또는 임대료
- 관리비, 유지보수비, 재산세 등 비용
- 공실률
- 대출 잔액 및 상환액
- 매각 계획 여부

검증:

- 임대 수입과 비용을 분리해서 받는다.
- 월세가 있는 자산과 거주 주택을 구분한다.
- 공실률은 0%에서 100% 사이여야 한다.

### 4. Pension and recurring income

입력 항목:

- 국민연금 또는 공적 연금 예상 수령액
- 퇴직연금, 개인연금, 보험성 연금
- 연금 개시 연령 또는 개시 월
- 물가연동 여부
- 배우자 또는 유족 급여 규칙
- 기타 반복 수입

검증:

- 월 지급인지 연 지급인지 명확히 선택한다.
- 세전/세후 금액 구분을 저장한다.
- 시작 월과 종료 월이 충돌하지 않는지 확인한다.

### 5. Savings and investments

입력 항목:

- 현금 및 예금
- 채권, 펀드, ETF, 주식 등 투자금
- 예상 수익률
- 배당 또는 이자 수입
- 인출률 또는 월별 인출 계획
- 위험 수준

검증:

- 예상 수익률은 시나리오 가정으로 분리한다.
- 원금 인출과 수익 수입을 구분한다.
- 투자 손실 가능성은 보수 시나리오에서 반영한다.

### 6. Assumptions review

사용자가 확인해야 할 기본 가정:

- 물가상승률
- 임대료 상승률
- 공실률
- 투자 수익률
- 세금 및 비용률
- 기대 수명
- 연금 개시 규칙

### 7. First projection

결과 화면은 다음을 제공한다.

- 월별 총수입
- 수입원별 breakdown
- 본인과 배우자의 해당 월 연령
- 필수 생활비 대비 충족률
- 수입 공백 구간
- 입력 신뢰도와 누락 정보
- 프로젝션 설명 및 다운로드

## Success metrics

- 첫 프로젝션 생성 완료율
- 필수 입력 대비 누락률
- 사용자가 기본 가정을 직접 검토한 비율
- 프로젝션 결과에서 설명 패널을 연 비율
- 사용자가 자산 정보를 갱신하는 빈도

## Edge cases

- 배우자가 없는 사용자
- 배우자 생년월일을 모르는 사용자
- 월세가 불규칙한 임대 자산
- 외화 수입 또는 해외 자산
- 연금 시작 월이 이미 지난 사용자
- 부동산 매각 계획이 있는 사용자
- 부채 상환이 수입보다 큰 기간이 있는 사용자
