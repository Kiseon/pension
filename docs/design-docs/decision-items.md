# Decision Items for Planning

노후 수입 설계 서비스를 구현하기 전에 결정해야 할 항목들이다. 각 항목은 제품 신뢰성, 사용자 이해도, 계산 정확도에 직접 영향을 준다.

## 1. Household modeling

- 현재 배우자만 지원할지, 과거 배우자/부양가족까지 확장할지
- 배우자 정보가 선택 입력인지 필수 입력인지
- 공동 소유 자산과 개인 소유 자산을 어떻게 구분할지
- 사망, 상속, 유족연금 이벤트를 MVP에서 포함할지

## 2. Projection horizon

- 기본 분석 종료 연령
- 사용자 지정 종료 연령 허용 여부
- 본인과 배우자 중 더 오래 사는 사람 기준으로 계산할지
- 월 단위 projection을 몇 년까지 지원할지

## 3. Income source taxonomy

- 국민연금, 퇴직연금, 개인연금을 별도 타입으로 둘지
- 임대료와 사업소득을 반복 수입으로 통합할지
- 배당, 이자, 투자 인출을 같은 금융수입 모델로 볼지
- 일회성 매각 수입과 반복 수입을 별도 cashflow로 분리할지

## 4. Real estate modeling

- 주택, 상가, 토지, 오피스텔 등 자산 유형 구분
- 월세, 보증금, 관리비, 유지보수비, 세금 처리
- 공실률과 임대료 상승률의 기본값
- 대출 원리금 상환과 금리 변동 지원 범위
- 매각 시점, 매각 비용, 양도세 모델링 여부

## 5. Financial asset modeling

- 예금/적금은 이자율 기반으로 계산할지
- 투자금은 기대수익률 기반인지 시나리오 기반인지
- 인출 전략: 정액, 정률, 필요액 보충, 사용자 지정
- 원금 소진 가능성을 허용하고 경고할지

## 6. Pension modeling

- 연금 개시 연령과 월 수령액 입력 방식
- 물가연동 여부
- 배우자 사망 시 유족연금 반영 여부
- 세전/세후 금액을 모두 입력받을지

## 7. Tax and fees

- MVP에서 세금을 단순 비율로 둘지, 세목별 모델링할지
- 임대소득세, 재산세, 종합소득세를 언제 반영할지
- 세후 현금흐름과 세전 현금흐름 중 기본 표시값
- 지역/국가별 규칙 확장 방식

## 8. Inflation and real value

- 명목 금액과 실질 금액을 모두 표시할지
- 기본 물가상승률 출처
- 수입원별 상승률을 개별 설정할지
- 현재가치 환산 기준월

## 9. Reliability harness

- 골든 시나리오의 첫 household 구성
- 결과 스냅샷 포맷
- assumption versioning 방식
- 계산 설명 lineage의 필수 필드
- 테스트 실패 시 release gate 정책

## 10. User experience

- 사용자가 모든 항목을 직접 입력하는지, 단계별 wizard를 제공할지
- 입력 불확실성을 어떻게 표시할지
- 월별 표, 연령별 차트, 수입원 breakdown의 우선순위
- "충분한 노후수입" 판단 기준을 제공할지, projection만 제공할지

## 11. Compliance and disclaimers

- 재무 자문이 아니라 계획 참고용임을 어떻게 명시할지
- 개인정보와 민감 금융정보의 보관 범위
- 사용자가 projection 근거를 내려받을 수 있게 할지
- 외부 금융/부동산 데이터 연동 시 동의와 철회 방식
